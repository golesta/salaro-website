Bringing the crispness back.

The build is in good shape, but it has drifted from the original system in five specific ways — all to do with contrast, restraint, and the navigation. Here is each one, with the exact fix to hand to Claude Code.

Fix 01 · Navigation

The selected tab is muddy, and it's on the wrong page.

Active = dark terracotta
+ 2px underline
Current page only
Hover = accent

Two bugs in one. First, the active link is a desaturated brown, not the crisp dark terracotta from the original — it reads as "slightly faded", not "you are here". Second, the active state is stuck on Home on every page (look at the Studio and Practice screenshots — "Home" is highlighted, not the page you're on).

You asked about an underline — yes. A 2px terracotta underline plus the colour change makes the current section unambiguous, and it echoes the hairline rules used elsewhere on the site.

Fix 02 · Readability

Long copy is set in faint italic serif. It's hard to read.

Italic serif = short ledes only
Body copy = upright sans
Darken body to 0.30 L
Never italicise paragraphs

This is the biggest one. The original system used italic Cormorant for short ledes and accent words only — one or two lines, set large. In the build it has spread to whole paragraphs of body copy (the entire founder bio on Studio, the team bios), in a light grey that sits at roughly 0.42 lightness.

Italic serif at small sizes in low-contrast grey is genuinely tiring to read — and it cheapens the editorial effect, because the italics stop meaning anything when everything is italic. The fix: body copy goes upright, in Inter Tight, darker. Keep one short italic serif line at the top of a section as the lede. That's the rhythm — italic to open, upright to explain.

/* Reserve italic display for ledes; body is upright sans, darker */ .lede { font-family: var(--font-display); font-style: italic; font-size: clamp(18px, 2vw, 22px); color: var(--ink-lede); } .body { font-family: var(--font-body); font-style: normal; font-size: 16px; line-height: 1.65; color: var(--ink-body); } /* New, higher-contrast text tokens — replace the old soft greys */ --ink-body: oklch(0.30 0.02 60); /* was 0.42 — long-form copy */ --ink-lede: oklch(0.38 0.022 60); /* short italic ledes */ --ink-label: oklch(0.48 0.02 60); /* was 0.58 — mono labels */
Fix 03 · Labels

The mono eyebrows are nearly invisible.

Darken to 0.48 L
Keep 0.22em tracking
Keep them small

The little uppercase mono labels ("UK WEB CONSULTANCY · PRACTISING SINCE 1995", "01 · FOUNDED", section eyebrows) are set so light they almost disappear on the cream. They're meant to be quiet, not invisible. One step darker fixes it without making them shout.

Fix 04 · Restraint

Lose the purple gradient washes.

Remove hero blob
Remove webshop wash
Paper stays flat

There's a faint purple-pink radial wash in the top-right of every inner-page hero, and the "content-managed webshop" band has a cream-to-pink gradient behind it. Both read as generic "AI SaaS" decoration — exactly the look we deliberately walked away from. The whole strength of this design is flat, warm paper with hairlines. No gradients. If a section needs separation, use --bg-warm as a flat fill (like the diagram band), never a gradient.

Fix 05 · Terracotta band

White-on-terracotta needs more contrast.

Text → near-white
Button border → cream
Phone number → full white

The "Let's take your business online" band and its "Start a conversation" button sit at low contrast — the outline button and the phone number almost vanish into the terracotta. Push the text to near-white and give the button a solid cream border so it reads as a button, not a ghost.
